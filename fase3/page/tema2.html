Listas y Enlaces HTML

Guía completa

Html [Image]. (2012). Recuperado de http://2.bp.blogspot.com/-SITChsTu2A4/UFuT_coD7MI/AAAAAAAADCI/8h2bXQVX28A/s640/html5.jpg

El lenguaje HTML define tres tipos diferentes de listas para agrupar elementos.

Listas ordenadas: los elementos están numerados u ordenados alfabéticamente. Para crear este tipo de listas se utiliza la etiqueta <ol>... </ol>. Las listas ordenadas son apropiadas cuando el orden exacto de los artículos es de importancia crítica para el significado del contenido. Ejemplo numerada:

  1. Lunes
  2. Martes
  3. Miercoles
Ejemplo alfabéticamente:
  1. Lunes
  2. Martes
  3. Miercoles

Listas desordenadas: los elementos no se presentan numerados ni ordenados alfabéticamente sino mediante símbolos como bolitas o cuadritos. Para crear este tipo de listas se utiliza la etiqueta <ul>... </ul> Este tipo de lista es el mejor usado para listas que no tienen un orden requerido. Los elementos de la lista aparecerán en cualquier orden en el que los codifique para el HTML, pero usted está determinando ese orden. Ejemplo con circulos:

  • Lunes
  • Martes
  • Miércoles
Ejemplo con cuadrados:
  • Lunes
  • Martes
  • Miércoles

Listas de definición: un conjunto de términos y definiciones similar a un diccionario, enciclopedias, etc...). Para estas listas utilizamos la etiqueta <dl>...</dl> .

Lo más importante de los documentos HTML son los enlaces. Ya que mediante los enlaces en HTML podemos comunicar una página con otra. De esta forma, enlazando documentos HTML podemos acabar tejiendo lo que es Internet. Para crear un enlace en HTML utilizamos el elemento A con la siguiente sintaxis:

<a> Contenido </a>

Pero solo con esto el enlace no tendrá mucha utilidad ya que el principal objetivo del enlace es enlazar a un destino. Para poder indicar el destino de un enlace utilizamos el atributo href. En valor del atributo href puede ser cualquier URI que represente un recurso. Es decir, una página, una parte de una página, una URL genérica, un archivo,… De esta forma el enlace en HTML lo crearemos con la sintaxis:

<a> href="url">Contenido </a>

Atributo target

_blank, el agente de usuario intentará abrir el enlace en una nueva ventana. La nueva ventana no tendrá nombre.

_self, el agente de usuario intentará abrir el enlace en el mismo marco donde está en código actual.

_parent, el agente de usuario intentará abrir el enlace en el frameset inmediatamente superior al que se encuentra la página. Esto suele suceder cuando tenemos el enlace en un área de frames.

_top, el agente de usuario intentará abrir el enlace en la ventana padre. En el caso de que exista un frameset lo eliminará y se hará con toda la ventana.

nombre_marco, se podrá indicar el nombre de un frame. En este caso el agente de usuario intentará abrir el enlace en el frame que coincida con el nombre. En el caso de no existir un frame con ese nombre lo abrirá en una nueva ventana, asignándole dicho nombre.

la sintaxis quedaría así:
<a href="enlace.html" target="_blank">enlace a abrir en una nueva ventana</a>

Atributo title

podemos describir de una forma textual el destino del enlace. Esto servirá no solo al usuario para que pueda obtener más información de dónde va, si no a las máquinas a la hora de asignar un nombre a la URI sobre la que estamos enlazando.

la sintaxis quedaría así: <a href="https://unad.edu.co" title="Universidad Nacional Abierta y a Distancia">Ir a unad Web</a>

Enlaces en la misma pagina

Podemos enlazar a una parte concreta de la pagina. Imagina tenemos varios título . Y lo que queremos hacer desde otro documento HTML o bien desde el mismo documento es enlazar directamente al inicio de cada título.

macamos el lugar que queremos enlazar, la sintaxis quedaría así:
<a name="partepagina" >contenido </a>

Creamos el enlace al lugar antes marcado, la sintaxis quedaría así:
<a href="#partepagina" >Ir a la parte de la pagina marcada</a>
